Weather App

Instructions

For this assignment, you'll be creating a weather application that allows users to search for current weather data by location.

Before you begin this assignment, you will need to create a free Open Weather API account.

After creating an account, visit the API key management page to obtain a key to use for this assignment.

Setup

The markup for the weather app is as follows:

Do NOT modify this index.html file. The included automated tests depend on this markup to run. All markup needed to complete this assignment is already included. There is also no need to write any CSS to complete this assignment; all required CSS for the assignment is already included.

There is no provided JavaScript for this assignment. Please write all of your code in the provided empty index.js file.

  1. The city and country code
  2. A Google Maps link to the location
  3. A weather icon representing the current weather condition
  4. A description of the current weather condition
  5. The actual temperature
  6. The perceived temperature
  7. The time the weather information was last updated.

Constructing the Open Weather URL

To obtain the weather data from the API, it will be necessary to construct the URL for the request using JavaScript. Here is the URL for Open Weather's current weather endpoint from their documentation (Note that the curly braces are not part of the URL, but intended to denote the places to replace portions of the URL with our own data):

https://api.openweathermap.org/data/2.5/weather?q={city name}&appid={API key}

We will need to add an extra parameter (units=imperial) to the query string to ensure we receive temperature data in fahrenheit instead of kelvins.

Here is an example of a completed URL:

https://api.openweathermap.org/data/2.5/weather?q=gainesville&units=imperial&appid=yourapikeyhere

The above URL is hardcoded to always retrieve data for Gainesville, but our app needs to query for a user-entered location. To fix this, we will need to use string concatenation in JavaScript to construct a new URL with each request. Assuming our user's query is stored in a variable named userQuery, our code might look something like this:

var weatherURL = "https://api.openweathermap.org/data/2.5/weather"
var queryString = "?units=imperial&appid=yourapikeyhere&q=" + userQuery
var fetchURL = weatherUrl + queryString

Constructing elements from the Weather API response

Most of the required data for this assignment can be pulled directly from the Open Weather API's response JSON. However, some elements are not obvious and require some additional insight.

First, the current weather endpoint does not return a full URL for the weather icon. Here's a portion of the response:

{
  "coord": {
    "lat": 29.6516,
    "lon": -82.3248
  },
  "weather": [
    {
      "id": 800,
      "main": "Clear",
      "description": "clear sky",
      "icon": "01d"
    }
  ],
  "dt": 1560350645,
  ...
}

The API only returns icon codes instead of full URLs. To construct the icon URL from the condition code, use the following URL from the Open Weather documentation:

https://openweathermap.org/img/wn/<icon-code-here>@2x.png

For the last updated time, use the dt property from the weather API response:

{
  "coord": {...},
  "weather": [...],
  "dt": 1560350645,
  ...
}

The time given in the response is a UNIX timestamp measured in seconds, however JavaScript timestamps are measured in milliseconds, so this number must first be multiplied by 1000. After converting to milliseconds, pass the value into JavaScript's Date constructor and then call toLocaleTimeString() on the resulting Date object.

var date = new Date(/* milliseconds go here */)
var timeString = date.toLocaleTimeString('en-US', {
  hour: 'numeric',
  minute: '2-digit'
})

Lastly, to create the google maps link (seen in figure 2), use the coord property from the API response:

{
  "coord": {
    "lat": 29.6516,
    "lon": -82.3248
  },
  ...
}

Then use those coordinates in the query parameter of the Google Maps URL also seen in figure 2:

https://www.google.com/maps/search/?api=1&query=29.6516,-82.3248
